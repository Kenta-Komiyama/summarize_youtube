概要（3-5行）
- DeepLearningAIがUdbと共同で提供する「Building Coding Agents with Two Executions」コースの紹介。講師はTereza TishkovaとFrancesco Zucchini。
- コースでは、エージェントがコードを生成・実行してタスクを遂行する仕組み、実行の安全対策（サンドボックス／コンテナ／MicroVM等）、および実践的な応用（データ分析、可視化、フルスタックアプリ生成）を学ぶ。
- 生成コードの力で未知のライブラリ利用や自動化が可能になる一方、ホストでの任意コード実行は危険なので、隔離とリソース制限が必須と強調している。

主要トピック（箇条書き）
- コーディングエージェントの基本概念：コード生成→実行を組み込む利点
- 実行の危険性と対策：ローカル実行、コンテナ化、MicroVMベースの安全なサンドボックス
- 実装演習：Pythonコードの生成・実行、ファイル入出力、チャットインターフェース
- 高度な応用：データアナリティクスエージェント（データ探索・プロット・要約）、Next.js を含むフルスタックアプリの自動生成
- 実戦でのベストプラクティス：リソース制限、ファイルシステム制御、実行環境の分離

重要ポイント（番号付き、セクション順で記載）
1. コード実行を許すエージェントはできることが格段に広がる（例：モデルが未知のPythonライブラリを見つけて簡潔に解決） — 効率化や自動化に強力（導入部）。
2. ただし、生成コードをそのままホストで実行するのは危険。意図しない操作や悪意ある処理のリスクがある（問題提起）。
3. 安全な実行方法の比較：ローカル実行／コンテナ化（Docker等）／MicroVMベースのサンドボックス（より強い隔離とリソース制御）— 選択はユースケースと安全要件次第（実行環境の説明）。
4. コース実習：最初にシンプルなコーディングエージェントを作成（Python実行、ファイル読み書き、チャットでのやり取り）— 基礎から学べる（実装パート）。
5. サンドボックス化の実践：HDBと表記される隔離環境（文字起こしでは"HDB"とある）が登場し、Python/JavaScriptの実行をリソース制限・ファイル制御下で行う手法を学ぶ（隔離環境の強化）。
6. 応用演習：データ解析エージェント（自動でデータを探索・プロット・サイズ推定）と、複数ファイルを編集してNext.jsウェブアプリをサンドボックス内で生成するフルスタック課題（応用と成果物の提示）。
7. 結論：コードを書いて実行できるエージェントは、単にツールを呼ぶだけのものより遥かに強力。だが必ず隔離と制限を入れて安全に運用することが重要（まとめ）。

アクション項目 / 役立つTips
- まずは小さなエージェントを作って、「生成→実行→検証」のワークフローを確立する。  
- 任意コード実行はホストではなく、必ず隔離された環境で行う（コンテナやMicroVMベースのサンドボックスを推奨）。  
- サンドボックスにはリソース制限（CPU、メモリ、タイムアウト）とファイルシステムのアクセス制御を設定する。  
- ネットワークアクセスは必要最小限に制限し、外部通信が不要ならオフにする。  
- 実行ログと監査記録を残して、どのコードが何をしたか追跡できるようにする。  
- データ解析や可視化をやらせる場合は、入力データの型検証とサンプルでの安全性チェックを行う。  
- フルスタック生成は段階的に：まず単一ファイル、次に複数ファイル、最後にフレームワーク（Next.js）全体という流れで検証する。  
- 既存ライブラリの探索力が高いので、依存関係やライセンスにも注意する（自動で使われるライブラリが想定外のライセンスを含む場合がある）。

英語 TL;DR
This DeepLearningAI course teaches how to build agents that generate and safely execute code (using sandboxes/containers/MicroVMs), with hands-on projects from simple Python agents to full-stack Next.js app generation. Safety (isolation, resource limits, auditing) is emphasized throughout.
